# 1_2. 본체/ 메인보드
    1. 메인보드
        - 컴퓨터에는 CPU, 파워 서플라이, DRAM, 그래픽 카드, SDD 등 수많은 부속 제품들이 들어있다. 각 부품들을 하나로 연결해주는 회로와 밖으로 신호를 보낼 수 있는 출력 포트를 가지고 있는 부품이다.

    2. CPU(Central Processing Unit) / 프로세서

        2_1. CPU 기능
             Fetch(인출) : 메모리상의 프로그램 카운터가 가르키는 명령어를 CPU로 인출하여 적재한다.
            
             Decode(해석) : 명령어의 해석. 이 단계에서 명령어의 종류와 타켓을 판단한다.

             Execute(실행) : 해석된 명령어에 따라 데이터에 대한 연산을 수행한다.

             Writeback(쓰기) : 명령어대로 처리 완료된 데이터를 메모리에 저장한다.

         2_2. CPU 특징

            - 한개의 CPU는 하나의 프로세스를 처리해준다.

            -시분할 방식
                하나의 CPU(프로세서)는 하나의 프로세스만 실행이 가능하다. 그럼에도 여러 프로세스를 동시에 실행시킬 수 있는 것은 CPU가 처리하는 시간을 잘게 쪼개 여러 프로세스에 배분하는 방식으로 동작하기 때문이다.
        
            - 스케줄링
                메모리에 있는 준비 상태의 프로세스 중 하나를 선택해 CPU자원을 할당 관리.

                * 스케줄링이 일어나는 경우
                    - 실행상태에서 대기상태로 전환될 때 - 비선점
                    - 실행상태에서 준비상태로 전환될 때 - 선점
                    - 대기상태에서 준비상태로 전환될 때
                    - 종료될 때

                * 스케줄링 종류
                    - 비선점 스케줄링
                        - 이미 할당된 CPU를 다른 프로세스가 강제로 빼앗아 사용할 수 없는 스케줄링 기법이다. 프로세스가 CPU 할당을 받으면 해당 프로세스가 완료될 때 까지 CPU를 사용한다.
                        - 일괄 처리 방식의 스케줄링이다.(공정하지만 긴급 응답을 요하는 작업에 좋지 않다.)

                        -비선점 스케줄링 종류
                            - FCFS (First Come First Service) : 준비 상태 큐에 도착한 순서에 따라 CPU를 할당하는 기법. 공평성은 유지되지만 짧은 작업이 긴 작업을, 중요한 작업이 중요하지 않은 작업을 기다리게 됨.
                            - SJF (Shortest Job First) : 실행 시간이 가장 짧은 프로세스에 먼저 CPU를 할당하는 기법. 가장 적은 평균 대기 시간을 제공하는 최적의 알고리즘이다.


                    - 선점형 스케줄링
                        - 하나의 프로세스가 CPU를 할당받아 실행 하고 있을 때 우선순위가 높은 프로세스가 CPU를 강제로 빼앗아 사용할 수 있는 기법이다.
                        - 선점으로 인한 많은 오버헤드(어떤 처리를 하기 위해 들어가는 간접적인 처리 시간 · 메모리 등)가 생긴다.
                        - 시분할 시스템에 사용하는 스케줄링이다.(긴급을 요하는 우선순위를 갖는 시분할 처리, 실시간 처리에 유용하다.)
                       
                        - 선점형 스케줄링 종류
                            SRT(Shortest Remaining Time) : 현재 실행 중인 프로세스의 남은 시간과 대기 큐에 프로세스의 실행기간이 가장 짧은 프로세스에게 CPU를 할당하는 기법(비선점 기법인 SJF 알고리즘의 선점 형태로 변경한 기법)이다.
                    
                대표적인 스케줄링 종류 중 하나가 선점형(PREPTIVE, 우선순위가 높은 프로세스 부터 처리)로 뽑히고 있다.
        
        2_3. 캐시 메모리

            - 코어 내 기억 장치는 메인 메모리(RAM)에서 데이터를 읽어 올 수 있지만 메인 메모리는 기억 장치 보다 상대적으로 속도가 느리기 때문에 병목현상(두 구성 요소의 최대 성능의 차이로 인해 한 구성 요소가 다른 하드웨어의 잠재 성능을 제한 하는 것)이 발생할 수 있다. 이 속도 차이를 극복하기 위해 사용되는 것이다.

            -L1~L3의 3개 캐시 메모리를 사용하며, 숫자가 작을수록 용량이 더 작고, 빠르다.
        
        2_4. 코어(Core)

            - CPU 역할을 수행하는 물리적인 단위.
            
            - 제어 장치 (CU, Control Unit)
                메모리에서 명령을 가져와 해독과 해독된 명령어 실행을 지시하는 핵심 장치이다.
            
            - 연산 장치 (ALU, Arithmethic Login Unit) 
                제어 장치의 지시를 받아 산술, 논리, 비트 연산 등의 실제 연산을 수행하는 장치이다.

                산술논리연산장치(ALU(Arithmetic and Logical Unit)) : 산술연산(+,-,*,/), 논리연산(and, or not) 수행

            - 기억 장치 (Register)
                제어, 연산 장치 등에서 사용하는 임시 기억 장치의 역할.

                    -기억 장치 종류
                        PC : 다음 인출(Fetch) 될 명령어의 주소를 가지고 있는 기억 장치
                        AC : 연산 결과 데이터를 일시적으로 저장하는 기억 장치
                        IR : 가장 최근에 인출된 명령어(현재 실행 중인 명령어)가 저장되어 있는 기억 장치
                        SR : 현재 CPU의 상태를 가지고 있는 기억 장치
                        MAR : PC에 저장된 명령어 주소가 사용되기 전에 일시적으로 저장되는 주소 기억 장치
                        MBR : 기억장치에 저장될 데이터 혹은 읽혀진 데이터가 일시적으로 저장되는 버퍼 기억 장치

    3. MEMORY
        CPU에 의헤 시행될 프로그램이 저장되는 곳으로 주 기억장치,보조 기억장치, 그리고 캐시 메모리가 있다.

            -메모리 종류
                - 주 기억장치(메인메모리) : 데이터 저장을 위한 회로. RAM, ROM이 있다.
                    빠른 접근 속도이지만 휘발성, 상대적으로 적은 용량에 높은 비용이라는 단점을 가지고 있다.
                    
                    주 기억장치의 종류
                        - RAM(Random Access Memory)
                            SRAM(Static RAM)
                                전원이 연결되어 있는 동안만 정보가 유지된다.
                                사용이 편리하고 읽기/쓰기 시간이 짧다.
                            DRAM(Dynamic RAM)
                                주기적으로 재 충전 해줘야 정보가 유지된다.
                                전력 소비가 적고 하나의 칩으로 대량의 정보 저장이 가능하다.
                        - ROM(Read Only Memory)
                            부트스트랩 로더(전원이 켜지면 시스템 부팅을 제일 먼저 자동으로 실행하는 프로그램이다.)

                - 보조 기억장치 : USB, SD 카드 등의 장치로, 저속, 대용량의 장치이다.
               
    4. 버스(bus)
    5. 다양한 입출력 인터페이스 카드
        -비디오카드(GPU), 모뎀 등

# 프로세스
    - 프로그램을 구동하여 프로그램 자체와 프로그램의 상태가 메모리 상에서 실행되는 작업단위(컴퓨터에서 연속적으로 실행되고 있는 컴퓨터 프로그램)를 지칭한다.
    - 디스크에 있는 프로그램이 메모리에 로드한다.

    - 프로세스의 메모리 구조
        Code 영역 : 사용자가 작성된 프로그램 함수들의 코드가 CPU에서 수행할 수 있는 기계어 명령 형태로 변환되어 저장되는 공간이다.
        Data 영역 : 전역 변수 또는 static 변수 등 프로그램이 사용하는 데이터를 저장하는 공간이다.
        Heap 영역 : 동적할당을 위한 메모리 영역이다.
        Stack 영역 : 호출된 함수의 수행을 마치고 복귀할 주소 및 데이터(지역변수, 매개변수, 리턴값 등)을 임시로 저장하는 공간이다.
    
    - 프로세스 상태
        New(생성) : 프로세스가 생성된 상태이다. OS 커널에 존재하는 Ready Queue에 올라가면 ready 상태가 된다.
        Ready(준비) : 프로세스가 CPU로부터 메모리 공간을 할당받길 기다리는 상태이다. 이때 프로세스 스케줄러에 의해 프로세스가 할당을 받게 되면 Running 상태가 된다.
        Running(실행) : 명령어들이 실행되는 상태이다.
        waiting(대기) : 특정 event가 발생하길 기다리는 상태이다.
        terminated(종료) : 프로세스가 실행을 끝마친 상태이다.

    - PCB (Process Control Block)
        각 프로세스는 운영체제에서 프로세스 제어 블록(PCB)에 의해 표현된다.
        각 프로세스는 하나의 PCB를 가지며 PCB에는 해당 프로세스에 대한 모든 정보, 상태, CPU 사용시간, Process id 등을 저장하고 있다.
        
# 스레드 (Thread)
    - 작업을 처리하는 논리적인 단위라고 한다. 
            
    -하나의 코어에 보통 하나의 스레드가 존재하는 것이 일반적이지만 하나의 코어에 두개의 스레드(하이퍼 스레딩 기술)을 이용해 성능을 높이는 기술도 존재한다.

    -스레드 동기화 : 여러 스레드가 한 리소스를 사용하려 할 때 사용하려는 스레드 하나를 제외한 나머지들을 사용하지 못하도록 막아주는 일.

# 컴파일
    - 작성한 코드(컴퓨터 언어)를 실행파일(바이너리)로 변환하는 과정이다.
    - 컴파일 과정은 4가지 단계(전처리 과정 - 컴파일 과정 - 어셈블리 과정 - 링킹 과정)으로 나누어 진다.
        1. 전처리 과정: 전처리기를 통해 소스 코드 파일(*.c)을 전처리된 소스 코드 파일(*.i)로 변환하는 과정이다.
            - 주석 제거 : 소스 코드에서 주석을 전부 제거한다.
            - 헤더 파일 삽입 : #include 지시문을 만나면 해당하는 헤더 파일을 찾아 헤더 파일에 있는 모든 내용을 복사해서 소스 코드에 삽입한다.
            - 메크로 치환 및 적용 : 매크로 이름을 찾아서 정의한 값으로 전부 바꿔준다.

        2. 컴파일 과정: 컴파일러를 통해 전처리된 소스 코드 파일(*.i)을어셈블리어(사람이 이해할 수 있게 부호화한 것, (*.s)) 파일로 변환하는 과정이다.
            언어의 문법 검사가 이루어진다.
        
        3. 어셈블리 과정: 어셈블리어 파일(*.s)을 오브젝트 파일(*.o)로 변환하는 과정이다.
        4. 링킹 과정: 링커를 통해 오브젝트 파일(*.o)들을 묶어 실행 파일로 만드는 과정이다.
            이 과정에서 오브젝트 파일들과 프로그램에서 사용하는 라이브러리 파일들을 링크하여 하나의 실행 파일을 만든다.
            
# OS(운영체제)
    하드웨어를 잘 동작할 수 있도록, 여러 프로그램들이 최적의 성능을 낼 수 있도록 메모리나 CPU를 사용해서 프로그램을 구동시켜줌.

# DISK와 MEMORY 차이
    - 메모리
        컴퓨터의 CPU가 처리하는 데이터와 프로그램을 저장한다.
        데이터가 일시적으로 저장되는 휘발성 메모리이므로 데이터가 변경되거나 손실 될 수 있다.
        용량이 작은 대신 빠른 속도로 데이터를 저장하고 빼낼 수 있다.

    - 디스크
        사용자가 사용하고자 하는 데이터와 프로그램을 저장한다.
        속도가 매우 느린 대신 가격당 데이터 저장용량이 크고 전원이 인가되지 않아도 데이터가 날아가지 않는 비휘발성이기 때문에 용량이 큰 데이터를 안전하게 보관하는 역할을 한다.

    - 실행하려는 명령이 메인메모리에 전달되고 그 명령에 따라 디스크에서 저장된 프로그램을 메인메모리로 보낸다.

# 가상 메모리(Virtual Memory)
    실제 메모리 크기와 상관없이 메모리를 이용할 수 있도록 가상의 메모리 주소를 사용하는 방법.

# 디스크 스케줄링
    저장된 데이터를 어떻게 효율적으로 가져올 지에 대한 관리 방식.

# RAID
    - 여러 개의 디스크를 배열하여 속도, 안정성, 효율성, 가용성의 증대를 하는데 쓰이는 기술.
    - 저렴한 디스크를 여러개를 배열하여 대용량의 디스크로 제공.
    - RAID 0 ~6까지 종류와 구성방식이 다양하다.

# 장애 허용성 HT / HA

    - 내결함성(Fault Tolerance) : 활성 작업 중에 오류가 발생하면 해당 작업의 올바른 종료 상태가 출력.
    - 고가용성(High Availability) : 시스템의 성능이 부정적인 영향을 받지 않도록 할 뿐만 아니라 다운 타임을 제한하도록 설계.

 # ASP(Application Service Previder)

    - 애플리케이션 서비스 제공자 : 고가의 하드웨어, 소프트웨어를 도입하지 않고도 네트워크 인프라를 이용하여 다양한 솔루션을 사용할 수 있는 임대 서비스.
    - 대표적으로 SAAS가 있다.

# DRS(Disaster Recovery system)
    - 장애 복구 시스템

# 유틸리티 컴퓨팅(Utility Computing)
    - 컴퓨팅 자원을 필요한 만큼 사용하고 그 비용을 지불 하는 방식.

# P2P(Peer to Peer)
    - 서버를 거치지 않고 클라이언트 컴퓨터끼리 직접 통신하는 방식
    - 대표적으로 토렌트, 소리바다 등이 있다.

# 클라우드 컴퓨팅(Cloud Computing)
    - 데이터를 클라우드에 연결된 다른 컴퓨터로 처리하는 기술을 의미. 
    - 비용, 시간, 편리함까지 극적인 장점을 가지고 있음.

    - 멀티 클라우드

# CPU의 속도
    Hz(헤르츠) : 프로그램을 처리하는 속도
    CPU의 경우 계산 처리 시간을 감안해서 클럭 수가 정해지는데, 클럭 수가 높을 수록 CPU의 성능이 높다.

    클락 : CPU 내부에서 일정한 주파수를 가지는 신호로, 이 신호에 동기화되어 CPU의 모든 명령어가 동작하게 된다.


# 동기와 비동기

    1. 동기(동시에 일어나는) : 요청과 결과가 동시에 일어나야 함.
        - 순서에 맞춰 진행되는 장점이 있지만, 여러 가지 요청을 동시에 처리 할 수 없다.
         - 실제 CPU가 느려지는 것은 아니지만 시스템의 전체적인 효율이 저하된다.
    
    2. 비동기( 동시에 일어나지 않는) : 요청과 결과가 동시에 일어나지 않음.
         - 하나의 요청에 따른 응답을 즉시 처리하지 않아도, 다른 요청에 대해 처리 가능한 방식이다.
        - 여러 개의 요청을 동시에 처리 할 수도 있지만 동기 방식보다 속도가 떨어질 수 있다.
        - 비동기식 처리를 요청 할 때 할일이 끝난 후 처리 결과를 알려주는 콜백(Call Back) 함수로 결과값을 받아 불러온다.

    3. 제어권 : 제어권은 자신(함수)의 코드를 실행할 권리 같은 것이다. 제어권을 가진 함수는 자신의 코드를 끝까지 실행한 후, 자신을 호출한 함수에게 돌려준다.
     
        3_1. 블록킹(Blocking)
            자신의 작업을 진행하다가 다른 주체의 작업이 시작되면 다른 작업이 끝날 떄 까지 기다렸다가 자신의 작업을 시작한다.

        3_2. 논 블로킹(Non-Blocking)
            다른 주체의 작업에 관련없이 자신의 작업을 한다.

    4. 동기 & 비동기 와 블로킹 & 논블로킹 

        동기 + 블로킹 : 요청받은 함수의 작업이 끝나야 제어권을 돌려받음 + 요청자는 결과가 나올 때까지 계속 확인.
        동기 + 논블로킹 : 제어권은 바로 돌려줌 + 요청자는 결과가 나올때까지 계속 확인
        비동기 + 블로킹 : 다른 작업이 끝날 때를 기다려야 하기 때문에 동기 + 블로킹과 비슷한 작업 효율이 나온다.
        비동기 + 논블로킹 : 제어권은 바로 돌려줌 + 결과는 요청받은 함수가 알아서 알려줌

# 자료 구조의 종류
    1.선형 구조
        - 배열 (Array) : 가장 기본적인 데이터 구조. 셀의 수가 고정되고, 각 셀에는 인덱스 번호가 부여된다. 배열을 활용 시 부여된 인덱스를 통해 해당 셀 안에 있는 데이터에 접근이 가능하다.
            
            특징
                - 주로 동일한 데이터 유형을 가진다.
                - 배열의 각 요소에 좁근하는 시간은 O(1)로 모두 동일하다.
                - 연속된 메모리에 단일 블록화하여 데이터를 저장하지만 큰 배열일 경우, 필요 메모리 할당이 불가능할 수 있다.
                - 실제 메모리 상에서 물리적으로 데이터가 순차적으로 저장되기 때문에 데이터에 순서가 있으며, 인덱스가 존재하여 인덱싱(인덱스를 사용해 특정 요소를 리스트로부터 읽어 드리는것) 및 슬라이싱(요소에 특정 부분을 따로 분리해 조작하는 것이 가능하다.

        - 스텍 (Stack) : 순서가 보존되는 선형 데이터 구조. 가장 최근 요소 부터 처리하는 LIFO(Last In First Out) 메커니즘이다.

            특징
                -배열과 달리 스택은 상수 시간에 i번째 항목에 접근 할 수 없다.
                -스택에서 데이터를 추가하거나 삭제하는 연산은 상수 시간에 가능하다.
                -배열처럼 원소들을 하나씩 옆으로 밀어 줄 필요가 없다.

        - 큐 (Queue) : 가장 먼저 입력된 요소를 처리하는 FIFO(First In First Out) 메커니즘이다.

            특징
                - 큐는 양쪽으로 뚫려 있는 구조이기 때문에 한쪽에서는 데이터 삽입만 이루어지고 다른 한족은 데이터 추출만 진행된다.
                - 삭제 연산만 수행되는 곳을 프론트(front), 삽입 연산만 이루어지는 곳을 리어(rear)라고 한다.
                - 데이터 삽입 작동을 enQueue라고 하며 반대로 데이터 추출 작동을 deQueue라고 한다.

            apppend(): 오른쪽에서 데이터를 삽입
            appendleft() : 왼쪽에서 데이터를 삽입
             pop() : 오른쪽에서 데이터를 삭제
             popleft() : 왼쪽에서 데이터를 삭제


        - 덱 (Deque) : 양쪽에서 넣기와 빼기를 할 수 있는 일반화된 선형 구조이다.
            특징
                - 큐는 뒤로 넣고 앞으로 빼는 자료구조라고 한다면, 덱은 앞으로도 뒤로도 넣을 수 있고, 앞으로도 뒤로도 뺄 수 있는 자료구조이다.
                - 한 방향으로 넣고, 양방향으로 뺄 수 있는 구조이기에 스택과 큐의 특성을 모두 갖는, 혹은 스택과 큐를 조합한 형태의 자료구조이다.
            
            Scroll : 입력 제한 덱(입력이 한쪽에서만 발생하고, 출력은 양쪽에서 일어날 수 있음)

            shelf : 출력 제한 덱(입력은 양쪽에서 일어나고, 출력은 한쪽에서 일어나도록 제한)


        - 연결 리스트(Linked List) : 각 요소는 노드라는 것에 저장되는데, 노드의 연결로 이루어진 자료 구조.

            특징
                - 데이터들이 메모리 상에서 연속적인 공간에 저장되는 특징이 있다.
                - 메모리의 각 부분에 데이터들이 떨어져서 저장되어있고, 각 데이터마다 다음 순서의 데이터가 뭔지 알려주는 포인터가 존재하는 구조로 되어있다.

        - 이중 연결 리스트(Doubly Linked List)
            - 하나의 데이터와 두개의 포인터로 이루어진 노드들로 구성되어 있습니다. 이중 연결리스트를 사용하게 된다면 이전 노드를 가르키는 포인트가 추가되면서 이전 노드에 대한 정보를 알 수 있게 된다. 이로써 tail 노드에서 부터 시작해 탐색하는 과정이 가능해져 시간복잡도를 줄일 수 있다.


        - 해시 (Hash) : 해시 테이블, 해시맵, 해시표는 컴퓨터에서 키를 값에 매핑할 수 있는 구조인, 연관 배열 추가에 사용되는 자료 구조이다. 해시 테이블은 해시 함수를 사용하여 인덱스를 버킷(bucket)이나 슬롯(slot)의 배열로 계산한다.

        기본적으로 선형 구조로 구현되지만, collision 등의 문제를 해결에 따라서 비선형 구조로 구현되기도 한다.

    2. 비선형 구조
        - 그래프 (Graph) : 노드와 노드를 연결하는 간선(edge)을 하나로 모아 놓은 자료 구조 라고한다.

            특징
                - 네트워크 모델이다.
                - 2개 이상의 경로가 가능하다 -> 노드들 사이에 무방향/방향에서 양방향 경로를 가질 수 있다.
                - 부모-자식의 관계가 아니다.
                - 그래프는 크게 방향 그래프와 무방향 그래프가 있다.

            그래프의 유형
                - 무방향 그래프 : 서로 대칭적인 관계를 연결해서 나타낸 그래프
                    간선(edge)에 방향이 없는 그래프, 무방향 그래프의 간선은 간선을 통해서 양 방향으로 갈 수 있다.

                - 방향 그래프 : 간선을 화살표로 표현, 간선에 방향성이 존재하는 그래프이다.

        - 트리 (Tree) : 노드로 구성된 계층적 자료구조. 최상의 노드를 만들고, 로트 노드의 child를 추가하고 또 추가하는 방식이다.

# 선형 배열(Linear Arrays)
    - 데이터들이 선처럼 일렬로 늘어선 형태를 말한다. 파이썬에서는 여러가지 데이터들을 줄지어서 나열시킬 수 있는 리스트(list)가 있다.

        1. 리스트 길이에 관계없이 실행 속도가 빠른 연산
            .append() : 데이터를 리스트에 덧붙일 때 사용된다. 추가된 데이터는 리스트 맨 뒤에 자리 하게 된다.
            .pop() : 리스트 맨 끝에 있는 데이터를 밖으로 빼올 때 사용할 수 있습니다.

        2. 리스트 길이에 비례해서 실행되는 연산
            .insert() : 리스트에 원한느 데이터를 원하는 자리에 넣을 수 있다.
            del : 리스트에서 삭제하고 싶은 데이터를 데이터의 인덱스 넘버를 사용해서 삭제할 수 있다.

    장/단점
        장점 : 인덱스로 접근할 수 있기 때문에 접근 속도가 매우 빠르고 연속된 메모리 공간에 존재하기 때문에 관리하기가 편하다.

        단점 : 
            - 배열을 이용해 구현하기 때문에 배열이 갖고 있는 메모리 사용의 비효율성문제를 그대로 가지고 있다.
            ex) 배열의 크기 >> 데이터 수 : 메모리 공간의 낭비를 가져온다.
            - 삽입, 삭제 연산 후에 연속적인 물리 주소를 유지하기 위해 원소들을 이동시키는 추가 작업과 시간이 소요된다.
